🔍 기술 설명 및 비교

1. Vanilla JS (순수 자바스크립트)

특별한 라이브러리나 프레임워크 없이 브라우저 내장 API만으로 구현하는 방식입니다.
특징: 가장 빠르고 가볍지만, 코드량이 많아질 수 있고 DOM(화면 요소)을 직접 하나하나 조작해야 해서 복잡한 대규모 앱에는 불편할 수 있습니다.
핵심 코드: document.getElementById('id').value로 값을 가져오고, innerText로 화면을 갱신합니다.

2. jQuery (제이쿼리)

복잡한 자바스크립트 코드를 짧고 쉽게 쓸 수 있게 해주는 라이브러리입니다. (과거 표준처럼 많이 쓰임)
특징: $ 기호 하나로 요소를 쉽게 찾고 이벤트를 연결할 수 있습니다. 하지만 요즘은 브라우저 표준 기능이 좋아져서 사용이 줄어드는 추세입니다.
핵심 코드: $('#id').val()로 값을 쉽게 가져오고, .text()로 화면을 수정합니다.

3. Vue.js (뷰) & React (리액트) 개념 비교

Vue와 React는 현대적인 SPA(Single Page Application) 프레임워크입니다. 화면의 데이터가 바뀌면 알아서 화면을 고쳐주는 '반응형(Reactivity)' 시스템이 핵심입니다.

  • Vue.js의 특징:
    HTML(Template), JS(Logic), CSS(Style)가 한 파일(.vue)이나 명확히 분리된 구조로 되어 있어 배우기 쉽고 직관적입니다.
    위 예제처럼 v-model을 쓰면 변수와 입력창이 자동으로 양방향 연결되어, 값이 바뀌면 즉시 화면도 바뀝니다.
  • React(리액트)의 개념 (MVVM 관점):
    React도 Vue와 마찬가지로 MVVM(Model-View-ViewModel) 패턴의 철학을 공유합니다.
    1. 데이터 바인딩: 화면에 보여질 데이터는 'State(상태)'라는 곳에 저장됩니다.
    2. 자동 화면 갱신: 이 State 값이 바뀌면, 리액트 엔진이 이를 감지하고 변경된 부분만 자동으로 화면(View)에 반영해줍니다.
    즉, 개발자가 직접 document.getElementById로 값을 넣을 필요 없이, "데이터만 바꾸면 화면은 알아서 바뀐다"는 것이 핵심입니다.
